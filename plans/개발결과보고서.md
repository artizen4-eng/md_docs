## 1. 프로젝트 개요

### 1.1 배경

LLM(Large Language Model)이 출력하는 결과물은 대부분 Markdown 형식입니다. 사용자는 이를 Word 문서로 변환할 때 수동으로 서식을 정리해야 하는 번거로움이 있었습니다.

### 1.2 목표

LLM 결과물(Markdown)을 Word 친화적 일반 문서로 자동 변환하는 웹 애플리케이션 개발

### 1.3 핵심 가치

- **복사-붙여넣기 경험**: 클릭 한 번으로 Word에 바로 붙여넣을 수 있는 문서 생성
- **설정 가능성**: 프리셋 시스템으로 문서별 스타일 커스터마이징
- **다중 기기 접근**: 웹 앱으로 어디서든 사용 가능

## 4. 주요 기능

### 4.1 실시간 변환

- Markdown 입력 시 즉시 문서로 변환
- 미리보기 영역에서 실시간 결과 확인

### 4.2 프리셋 시스템

| 프리셋명 | 용도 | 특징 |
|---------|------|------|
| **보고서 기본** | 표준 보고서 | 폰트 11pt, 줄간격 1.5 |
| **회의록** | 회의록 작성 | 폰트 10pt, □ 불릿 |
| **간단한 메모** | 메모 작성 | 간소화된 서식 |

### 4.3 커스터마이징

- 헤딩 스타일 (크기, 볼드, 간격)
- 본문 스타일 (크기, 줄간격, 자간, 문단 간격)
- 리스트 스타일 (불릿, 들여쓰기)
- 코드 처리 (인라인, 블록)
- 인용문 스타일
- 테이블 형식

### 4.4 내보내기

- **클립보드 복사**: Navigator Clipboard API 사용
- **Word 파일**: HTML → DOCX 변환 후 다운로드

### 4.5 데이터 영속화

- LocalStorage에 사용자 프리셋 자동 저장
- 브라우저 새로고침 시 설정 유지

## 7. 구현 상세

### 7.1 변환 규칙

| 마크다운 | 변환 결과 |
|---------|----------|
| `# 제목 1` | 제목 1 (크기 20, 볼드) |
| `## 제목 2` | 제목 2 (크기 16, 볼드) |
| `### 제목 3` | 제목 3 (크기 14, 볼드) |
| `**볼드**` | 볼드 |
| `*이탤릭*` | 이탤릭 |
| `- 리스트` | • 리스트 |
| `1. 순서` | 1. 순서 |
| `` `코드` `` | 텍스트 (백틱 제거) |
| ` ```코드블록``` ` | 제거 또는 인용문 |
| `> 인용` | │ 인용 |

### 7.2 상태 관리 (Zustand)

```typescript
interface ConverterStore {
  // 상태
  currentPreset: ConverterPreset;
  customPresets: ConverterPreset[];
  inputMarkdown: string;
  outputDocument: string;

  // 액션
  setInputMarkdown: (markdown: string) => void;
  setOutputDocument: (document: string) => void;
  setCurrentPreset: (preset: ConverterPreset) => void;
  addCustomPreset: (preset: ConverterPreset) => void;
  updateCustomPreset: (id: string, preset: Partial<ConverterPreset>) => void;
  deleteCustomPreset: (id: string) => void;
  resetToDefault: () => void;
}
```

### 7.3 핵심 모듈

| 파일 | 역할 |
|------|------|
| [`documentConverter.ts`](src/lib/transformers/documentConverter.ts) | Markdown → 일반 텍스트 변환 |
| [`converterStore.ts`](src/store/converterStore.ts) | Zustand 상태 관리 + persist |
| [`useConverter.ts`](src/hooks/useConverter.ts) | 실시간 변환 Hook |
| [`clipboard.ts`](src/lib/exporters/clipboard.ts) | 클립보드 복사 (fallback 포함) |

## 13. 결론

본 프로젝트는 LLM 결과물인 Markdown을 Word 친화적 문서로 변환하는 웹 애플리케이션을 성공적으로 개발하였습니다.

React + TypeScript + Vite 기반의 현대적인 기술 스택을 사용하여 빠르고 안정적인 애플리케이션을 구현하였으며, Zustand를 활용한 상태 관리와 LocalStorage를 통한 데이터 영속화로 사용자 경험을 개선했습니다.

Cloudflare Pages를 통한 배포로 전 세계 어디서든 빠르게 접근 가능하며, 앞으로도 지속적인 개선을 통해 사용자 편의성을 높여나갈 계획입니다.
